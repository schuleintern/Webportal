109 2.1.0-alpha1 (2014-09-15)
110 -------------------------
111 
112 * Changed: Copied most of the header-semantics for the PSR draft for
113  representing HTTP messages. [Reference here][psr-http].
114 * This means that `setHeaders()` does not wipe out every existing header
115  anymore.
116 * We also support multiple headers with the same name.
117 * Use `Request::getHeaderAsArray()` and `Response::getHeaderAsArray()` to
118  get a hold off multiple headers with the same name.
119 * If you use `getHeader()`, and there's more than 1 header with that name, we
120  concatenate all these with a comma.
121 * `addHeader()` will now preserve an existing header with that name, and add a
122  second header with the same name.
123 * The message class should be a lot faster now for looking up headers. No more
124  array traversal, because we maintain a tiny index.
125 * Added: `URLUtil::resolve()` to make resolving relative urls super easy.
126 * Switched to PSR-4.
127 * #12: Circumventing CURL's FOLLOW_LOCATION and doing it in PHP instead. This
128  fixes compatibility issues with people that have open_basedir turned on.
129 * Added: Content negotiation now correctly support mime-type parameters such as
130  charset.
131 * Changed: `Util::negotiate()` is now deprecated. Use
132  `Util::negotiateContentType()` instead.
133 * #14: The client now only follows http and https urls.
